# Support

Use this file to decide where to ask for help and what to include.

## Before opening an issue

Start with:
- [Project overview](README.md)
- [Getting started](QUICKSTART.md)
- [Tiny mode](docs/tiny/README.md)
- [Self-hosted mode](docs/self-hosted/README.md)

## Which issue should I open?

- runtime or product bug: bug report
- docs problem: docs report
- feature idea: feature request

Include:
- whether you are using Tiny mode or self-hosted mode
- version or commit
- install method
- exact repro steps

## Security

Do not report vulnerabilities in public issues.

Use the [security policy](SECURITY.md).
